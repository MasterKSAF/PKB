# UI/UX

Рабочая зона команды UI/UX в проекте `PKB_neuroassistant`.

Эта папка нужна как входная точка по материалам UI/UX-направления: версии фронтенда, инструкции, чек-листы и MEMO встреч.

## Текущая ветка

Ветка: `feature/ui-import-frontend-v1`

Базовая рабочая версия интерфейса на этой ветке: `frontend-v1`.

`frontend-v1` временно используем как основную версию для запуска, проверки, обсуждения с командой и дальнейшей доработки.

## Что здесь лежит

| Папка | Что внутри |
|---|---|
| `frontend-v1/` | Первая рабочая версия фронтенда. Внутри есть свой подробный README: запуск, Docker, структура файлов, сценарии, backend-контракт. |
| `frontend-v2/` | Место под альтернативную версию интерфейса. |
| `docs/` | Инструкции, workflow, роли, чек-листы и вспомогательные материалы команды. |
| `meetings/` | Материалы встреч по датам: MEMO в Markdown и ссылки на видеозаписи. |

## Куда смотреть в первую очередь

1. Для запуска UI V1: `frontend-v1/README.md`.
2. Для правил работы с GitHub: `docs/workflow.md`.
3. Для ролей в команде: `docs/roles.md`.
4. Для проверки интерфейса по ТЗ: `docs/qa-checklist.md` и актуальные review/checklist-файлы в `docs/`.
5. Для итогов встреч: `meetings/<дата>/meeting-memo-<дата>.md`.

## Как работаем

1. Не работаем напрямую в `main`.
2. Общая рабочая ветка команды — `develop`.
3. Каждый участник создает свою ветку от `develop`.
4. Изменения заливаются через Pull Request в `develop`.
5. В `main` попадает только проверенная и согласованная версия.

## Что не коммитим

- `node_modules`;
- `dist`;
- временные файлы;
- локальные `.env`;
- большие видеофайлы;
- скриншоты и черновики без необходимости.

Видео встреч храним вне GitHub, например на Яндекс.Диске или Google Drive. В репозитории оставляем только markdown-файл со ссылкой.
