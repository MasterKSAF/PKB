# Ответы заказчика по открытым вопросам UI/UX

Файл фиксирует ответы заказчика, полученные после встречи UI/UX от 04.05.2026.

Источник ответов: [таблица на Яндекс.Диске](https://disk.yandex.ru/edit/d/9kxg605YrKFHU5TRco2knCPegnqahzm72s0qoIz-cKg6MVgtUGR0b2hlZw).

## Решения и выводы для UI

| Вопрос | Ответ заказчика | Статус | Вывод для UI |
|---|---|---|---|
| Какая версия ТЗ является основной для UI/UX? | Рабочей считать объединенную версию. Если между версиями есть противоречия, их нужно показывать конкретно. Главный ориентир — что команда успевает сделать. | Частично закрыт | Работаем от объединенной версии ТЗ. Противоречия выносим отдельным списком, если они влияют на MVP. |
| Нужен ли расчетный модуль в MVP? | Расчетный модуль пока исключен. | Закрыт | Убираем расчетный модуль из текущего MVP и оставляем как возможный будущий этап. |
| Какие роли и права нужны, нужен ли отдельный экран авторизации? | Отдельный экран авторизации нужен. Роли пока: администратор и пользователь. | Закрыт | В UI оставляем две роли: пользователь и администратор. Добавляем экран входа перед основной рабочей областью. |
| В каком формате открывать документы? | В том формате, в котором документ хранится в базе знаний. Главное — не открытие одного документа, а получение ответа с цитатами из разных документов. | Частично закрыт | UI показывает страницу или документ в том представлении, которое отдаёт backend. Отдельную конвертацию в PDF не закладываем как требование MVP. |
| Какие UI-библиотеки допустимы? | Заказчик не решает технические вопросы. | Закрыт | UI-библиотеку выбирает команда. Для текущей UI V1 оставляем существующий стек. |
| Как организовать историю диалогов? | Нужны разные срезы: по инженерам, проектам, темам, рабочим сессиям и, возможно, другие. Конкретная организация — технический вопрос. | Частично закрыт | В UI добавляем историю диалогов с фильтрами и возможностью открыть или продолжить найденный диалог. |
| Нужен ли поиск по чатам? | Крайне желателен для сбора статистики и формулировки ответов на частые вопросы. Нужна конкретика реализации. | Частично закрыт | Разделяем поиск внутри активного чата и поиск по прошлым диалогам в истории. |

## Открытые уточнения

1. Нужно подготовить конкретный список противоречий в версиях ТЗ, если они влияют на UI или MVP.
2. Нужно отдельно согласовать, какие срезы истории диалогов обязательны в MVP.
3. Нужно предложить конкретную модель поиска по чатам: активный чат, прошлые диалоги, частые вопросы, статистика.
