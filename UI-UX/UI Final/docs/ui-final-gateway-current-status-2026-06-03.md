# UI Final: статус интеграции с актуальным Gateway

Дата: 03.06.2026  
Ветка UI: `feature/ui-final-gateway-current`  
Источник UI: свежая версия `UI Final` после правок по объединенному чек-листу  
Источник Gateway: `origin/develop`, коммит `b8939d0`

## Что подключено в UI

- Авторизация и профиль: `POST /api/v1/auth/token`, `GET /api/v1/auth/me`.
- Чат: создание сессии, отправка сообщения, получение истории сессии, обратная связь.
- Поиск: основной путь `POST /api/v1/documents/search`, резервный путь `POST /api/v1/text/search`.
- База знаний: список документов, загрузка файла, повторная обработка OCR, дерево классификаторов.
- История: список чат-сессий, просмотр сообщений, экспорт истории.
- QA: получение метрик через `GET /api/v1/monitor/metrics`.
- Администрирование: список пользователей, аудит, обновление роли пользователя.
- Превью источников: страница документа и файл документа по `documents/{id}/pages/{page}/preview` и `documents/{id}/file`.

## Что оставлено на mock/fallback

- Вкладка `Проверка`: в актуальном Gateway нет явного контракта для сверки проектных параметров с требованиями НСИ.
- Полноценное управление проектами в дереве чатов: Gateway сейчас отдает чат-сессии, но отдельный контракт проектов не зафиксирован.
- Часть действий администрирования базы знаний: UI показывает сценарии, но реальные действия зависят от финального backend-контракта.

## Проверка UI

- `npm run lint` — успешно.
- `npm run build` — успешно.
- Demo-режим сохранен: если Gateway недоступен, интерфейс не падает и показывает демонстрационные данные.
- Prod-режим готов обращаться к `VITE_API_BASE_URL`, по умолчанию `http://127.0.0.1:8081/api/v1`.

## Блокер Gateway

Актуальный единый Gateway сейчас не стартует:

```text
ImportError: cannot import name 'router' from 'mocks.auth_service.main'
```

Причина: `backend/gateway_service/mocks/gateway.py` импортирует `router` из сервисов, а текущие сервисы экспортируют `app = FastAPI(...)`, а не `router`.

Пока это не исправлено на стороне Gateway, полноценный end-to-end smoke test UI через единую точку `8081` невозможен.
