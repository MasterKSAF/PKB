# Резюме встречи UI/UX от 25.05.2026

Запись встречи: [UI/UX meeting 2026-05-25](https://disk.yandex.ru/i/ytgB_bvBujjETA)

## Что обсуждали

1. Показали объединенный интерфейс на базе UI V1 с переносом сильных решений из версий Елены и Гульнары.
2. От UI V1 оставили основной каркас: чат, поиск, база знаний, проверка, история, QA, администрирование, роли и источники.
3. Из версии Елены взяли базу знаний по разделам, работу внутри проектов, дерево «Мои проекты» и вложенные чаты.
4. Из версии Гульнары взяли вход по ролям, светлую голубую тему, синий кораблик, расширенное preview, масштабирование и крупный просмотр документа.
5. Обсудили базу знаний, OCR-статусы, действия с документами и возможное меню документа.
6. Отдельно проговорили спорные места: вкладка «История», роли и права, QA-журналы, preview сложных документов и будущая стыковка с Gateway.

## Что решили и что делаем дальше

1. Объединенная версия выложена на GitHub для локальной проверки группой: [feature/ui-final](https://github.com/NeuronsUII/PKB_neuroassistant/tree/feature/ui-final).
2. Всем участникам скачать интерфейс, пройтись по вкладкам и подготовить чек-лист.
3. В чек-листе фиксировать, что убрать, что оставить и что добавить из версий Елены, Гульнары или UI V1.
4. Отдельно отметить сильные решения из альтернативных версий, которые еще стоит перенести в объединенный интерфейс.
5. За неделю собрать предложения, замечания и риски, затем сформировать один общий пакет правок.
6. Продолжить контроль адаптации UI к Gateway.
7. После выравнивания UI переходить к практической стыковке с Gateway/backend.

## Открытые вопросы

1. Оставлять ли отдельную вкладку «История» или часть сценариев перенести в дерево проектов и чатов?
2. Какие роли и права должны быть в финальной версии?
3. Нужны ли пользователю QA-журналы или отдельный механизм сообщения об ошибке?
4. Какие действия должны быть в меню документа в «Базе знаний»?
5. Нужен ли выбор LLM в «Администрировании»?
6. Когда выполнять первую фактическую проверку UI против Gateway/API?

## Итог

Объединенный интерфейс принят как рабочая база для проверки. Следующий шаг - чек-листы, общий пакет правок и переход к интеграции с Gateway.
