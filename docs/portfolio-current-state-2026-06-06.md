# Состояние портфельной сборки на 2026-06-06

## Назначение

Эта сборка подготовлена для личного репозитория `PKB`, на который можно давать ссылку из резюме. Цель — показать текущее состояние работы над нейроассистентом без лишних черновиков, личных папок участников и временных файлов.

## Что перенесено

| Блок | Что вошло |
| --- | --- |
| UI | `frontend/` — актуальная версия `UI Final` с подключением к Gateway и последними локальными правками по окну `База знаний`. |
| Gateway | `backend/gateway_service/` — рабочий gateway/mock-контур для локальной проверки UI. |
| Backend | Основные сервисы из свежего `develop`: auth, query, registry, orchestrator, parser, converter-validator, rag builder, rag search, integration, shared, service checker. |
| Docs | API-контракты, схемы, пайплайны, документация по UI Final, планы спринтов и отчёт по RAG Search coverage. |

## Что сознательно не перенесено

- Личные папки участников из общего репозитория.
- Записи встреч, видео, временные документы и экспериментальные файлы.
- Старые версии UI V1/V2 и ветки-spike.
- `node_modules`, виртуальные окружения, кеши, логи, `__pycache__`.

## Источники сборки

| Источник | Версия |
| --- | --- |
| Backend/docs | `NeuronsUII/PKB_neuroassistant`, `origin/develop`, commit `6b01f5e` |
| UI Final | `feature/ui-final-gateway-current`, commit `82138ef`, плюс локальные правки окна `База знаний` |
| Gateway | рабочий локально проверенный gateway-контур на базе `develop_gateway`, commit `cf6eeec` |

## Что сейчас можно показать

- Современный интерфейс инженера с авторизацией, ролями, чатом, поиском, базой знаний, историей, QA и администрированием.
- Архитектуру микросервисов и пайплайны обработки документов.
- API-документацию Gateway, Query, Registry, RAG и других сервисов.
- Локальный запуск UI и проверку связки UI + Gateway.

## Что ещё требует доработки

- Финальная стыковка UI с реальным backend вместо mock/dev-контуров.
- Стабильный Docker/Nginx-сценарий для серверного деплоя.
- Реальная база знаний, индексация и проверка качества RAG-ответов на эталонных вопросах.
- Финальная модель проектов, прав доступа, справочников и истории диалогов.
