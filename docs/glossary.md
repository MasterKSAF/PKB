# Глоссарий терминов и сокращений

## Классификаторы и коды

| Сокращение | Полное название | Описание |
|------------|----------------|----------|
| **МКС** | Межгосударственный классификатор стандартов (ICS — International Classification for Standards) | Иерархический классификатор для каталогизации стандартов. Код вида `31.240` (Электроника. Монтажные изделия). |
| **ОКСТУ** | Общесоюзный классификатор стандартов и технических условий | Классификатор стандартов СССР. Используется для документов советской эпохи (`era=USSR`). |
| **УДК** | Универсальная десятичная классификация (UDC — Universal Decimal Classification) | Международная система классификации документов. Код вида `621.3` или `669.1` |
| **ОКС** | Общероссийский классификатор стандартов | Российский аналог МКС. Используется для документов РФ (`era=RF`). |

## Типы документов

| Сокращение | Полное название | Описание |
|------------|----------------|----------|
| **ГОСТ** | Государственный стандарт | Национальный стандарт СССР / РФ |
| **ГОСТ Р** | Государственный стандарт Российской Федерации | Российский национальный стандарт |
| **ОСТ** | Отраслевой стандарт | Стандарт конкретной отрасли промышленности |
| **РД** | Руководящий документ | Нормативный документ методического характера |
| **ТУ** | Технические условия | Документ, устанавливающий требования к конкретной продукции |
| **ISO** | International Organization for Standardization | Международный стандарт |
| **DNV** | Det Norske Veritas | Нормы классификационного общества (судостроение) |
| **ASTM** | American Society for Testing and Materials | Стандарты ASTM (международные) |

## Технические термины

| Термин | Описание |
|--------|----------|
| **RAG** | Retrieval-Augmented Generation — подход к генерации ответов с использованием поиска релевантных фрагментов документов |
| **Секция (section)** | Логический фрагмент документа (раздел, параграф, таблица, рисунок, формула). Назначается Registry при сохранении карточки документа. Хранится в `registry.document_sections` |
| **Chunk (чанк)** | Единица поискового индекса (не более 512 токенов). Получается разбиением секции (одна секция типа `text` → несколько чанков; таблица/изображение/формула → один чанк). Хранится в `rag.document_chunks` |
| **Embedding (эмбеддинг)** | Векторное представление текста (массив чисел с плавающей точкой) для семантического поиска |
| **pgvector** | Расширение PostgreSQL для работы с векторными данными и косинусной близостью |
| **Longpoll** | Механизм асинхронного ожидания: клиент отправляет запрос и держит соединение открытым до получения ответа или истечения таймаута |
| **FSM** | Finite State Machine — конечный автомат, модель смены состояний документа |
| **Saga** | Паттерн распределённых транзакций: последовательность локальных транзакций с компенсационными действиями при сбоях |
| **SHA-256** | Криптографическая хэш-функция, используется для вычисления `file_hash_sha256` (контроль целостности файла) и `title_hash_sha256` (бизнес-ключ документа) |
| **MinIO** | S3-совместимое объектное хранилище для файлов документов |
| **CAS-путь** | Content-Addressable Storage — путь к файлу вида `{doc_id}/v{n}/{hash}.{ext}`, гарантирующий уникальность |
| **JWT** | JSON Web Token — формат токена для аутентификации |
| **RBAC** | Role-Based Access Control — управление доступом на основе ролей |
| **Circuit Breaker** | Паттерн защиты: при N последовательных ошибках вызов этапа блокируется на заданное время |
| **OCR** | Optical Character Recognition — оптическое распознавание текста |
| **LLM** | Large Language Model — большая языковая модель (используется для генерации ответов) |
| **Bbox** | Bounding box — координаты блока на странице: `[x1, y1, x2, y2]` в нормализованных единицах (0..1) |

## Идентификаторы

| Термин | Описание |
|--------|----------|
| **task_id** | UUID — временный идентификатор на этапах загрузки и preview (до создания карточки документа) |
| **document_id** | UUID — постоянный идентификатор документа, назначается Registry при создании карточки |
| **version_id** | UUID — идентификатор версии файла документа |
| **section_id** | BIGINT — идентификатор секции/раздела документа (sequence, назначается Registry) |
| **chunk_id** | BIGINT — идентификатор чанка (фрагмента для поиска) |
| **message_id** | UUID — идентификатор сообщения в чате |
| **session_id** | UUID — идентификатор сессии чата |

## Статусы документа

| Статус | Пайплайн | Описание |
|--------|----------|----------|
| `draft` | 1 | Черновик после загрузки файла |
| `uploaded` | 1 | Файл загружен в MinIO, ожидание preview |
| `previewing` | 1 | Выполняется preview-фаза |
| `awaiting_decision` | 1 | Preview завершён, ожидание решения пользователя |
| `parsing` | 1 | Выполняется полный OCR/Parser |
| `validation` | 1 | Конвертация и валидация |
| `ready_for_promotion` | 1 | Авто-валидация пройдена |
| `review_required` | 1 | Требуется ручное подтверждение |
| `approved` | 1 | Оператор подтвердил |
| `registry` | 1 | Документ записан в реестр |
| `pending_index` | 2 | Ожидание индексации |
| `indexing` | 2 | Выполняется индексация |
| `indexed` | 2 | Документ проиндексирован |
| `duplicate` | 1 | Документ-дубликат |
| `new_version` | 1 | Создана новая версия |
| `archived` | 1 | Документ архивирован |
| `failed` | 1/2/3 | Ошибка на одном из этапов |

## Форматы JSON-контейнеров

| Схема | Этап | Описание |
|-------|------|----------|
| `raw_ocr_v4` | OCR/Parser output | Сырой плоский JSON: блоки текста, таблиц, изображений с bbox и font-атрибутами |
| `validated_v3` | Converter-validator output | Иерархический JSON: секции, метаданные, ссылки, терминология |
| `registry_for_rag` | Registry → RAG Builder | Плоский JSON со списком секций, ссылками в БД и защищёнными spans |
