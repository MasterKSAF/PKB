# Описание окна «База знаний»

Окно «База знаний» в текущем интерфейсе объединяет управление документами и просмотр документов базы знаний.

## 1. Верхняя панель действий

В верхней части окна расположены три действия:

- **Загрузить документ**
- **Загрузить по ссылке**
- **Повторить OCR**

### Загрузить документ

Кнопка открывает выбор файла на компьютере пользователя.

После выбора файла в интерфейсе отображается сообщение:

> Выбран файл: [название файла]. В рабочем режиме он будет отправлен в хранилище, OCR и индекс.

В рабочем режиме файл отправляется через:

`POST /documents`

### Загрузить по ссылке

Кнопка отображает уведомление о загрузке документа по ссылке.

### Повторить OCR

Кнопка запускает повторную обработку OCR для документа.

В рабочем режиме используется запрос:

`POST /documents/{document_id}/reprocess`

## 2. Карточки состояния

Под верхней панелью отображаются четыре карточки состояния:

- **Документов в реестре**
- **OCR завершен**
- **Индексировано**
- **Требуют внимания**

Карточки считаются по списку документов, который загружается из:

`GET /documents`

## 3. Раздел «База знаний по разделам»

В этом блоке отображаются карточки разделов базы знаний.

Каждая карточка содержит:

- название раздела;
- описание раздела;
- количество документов в разделе.

Данные загружаются через:

`GET /classifiers/tree`

При нажатии на карточку раздел выделяется в интерфейсе.

## 4. Таблица документов

Ниже расположена таблица документов.

В таблице есть колонки:

- **Документ**
- **Тип**
- **Версия**
- **Источник**
- **OCR статус**
- **Индекс статус**
- **Обновлен**
- меню действий через три точки

Список документов загружается через:

`GET /documents`

Для каждого документа интерфейс использует поля:

- `document_id` или `id`;
- `title`, `filename` или `name`;
- `document_type`, `source_type` или `type`;
- `latest_version` или `version`;
- `source` или `uploaded_by`;
- `status`;
- `updated_at` или `created_at`.

## 5. Действия по документу

У каждого документа есть меню через три точки.

В меню доступны действия:

- **Открыть предпросмотр**
- **Скачать документ**
- **Перезапустить OCR**
- **Изменить метаданные**
- **История версий**
- **Назначить раздел**
- **Архивировать**

### Открыть предпросмотр

Открывает модальное окно предпросмотра документа.

В рабочем режиме используется запрос:

`GET /documents/{document_id}/file`

В окне отображаются:

- название документа;
- ID документа;
- тип;
- версия;
- OCR статус;
- индекс статус;
- ссылка на файл, если она вернулась от Gateway;
- карточка документа.

### Скачать документ

Запускает скачивание документа.

Сначала интерфейс получает данные файла через:

`GET /documents/{document_id}/file`

Если в ответе есть `file_url`, интерфейс пытается скачать файл по этой ссылке.

### Перезапустить OCR

Запускает повторную обработку выбранного документа.

Используется запрос:

`POST /documents/{document_id}/reprocess`

### Изменить метаданные

Открывает модальное окно редактирования метаданных.

В окне есть поля:

- название документа;
- тип;
- версия;
- источник.

После сохранения изменения отображаются в интерфейсе.

### История версий

Открывает модальное окно истории версий документа.

В рабочем режиме используется запрос:

`GET /documents/{document_id}/versions`

В окне отображаются:

- номер версии;
- ID версии;
- название;
- статус;
- дата создания;
- размер файла, если он есть.

### Назначить раздел

Открывает модальное окно выбора раздела базы знаний.

В окне отображается список разделов.

После выбора раздел отображается под названием документа в таблице.

### Архивировать

Открывает окно подтверждения архивирования документа.

В рабочем режиме используется запрос:

`DELETE /documents/{document_id}`

После подтверждения документ скрывается из таблицы.

## 6. Уведомления

В окне используются всплывающие уведомления.

Они показывают результат действий:

- выбран файл;
- файл отправлен в обработку;
- OCR запущен;
- документ скачивается;
- метаданные обновлены;
- раздел назначен;
- документ архивирован;
- действие не принято серверной частью.
