## API RAG Service (rag-service:8087)

Сервис векторного поиска, генерации ответов, индексации документов и вычисления embeddings.  
**Внутренний сервис.** Используется в двух режимах, соответствующих пайплайнам системы:

1. **Пайплайн индексации (Pipeline 2)**
   - **Режим A: RAG индексация** — чанкинг, вычисление embeddings, построение векторного индекса (**пишет БД**)
   - **Режим B: RAG поиск** — гибридный поиск и генерация ответа (**читает БД**)

**Базовый URL (внутренний)**: `http://127.0.0.1:8087/api/v1`

### Формат ответа

Успех — данные возвращаются напрямую.  
При ошибке: `{ "error": { "code": "INDEXING_FAILED", "message": "...", "details": {} } }`

---

## Режим A: RAG индексация (пишет БД)

Этот режим запускается после успешного завершения **Пайплайна 1 (Формирование документа)**.  
На вход получает JSON со ссылками в БД, на выходе — статус завершения индексации.

### POST /rag/index

Построение чанков, вычисление embeddings и индексация документа.  
Вызывается Orchestrator после завершения Пайплайна 1.

**Процесс внутри:**

| Шаг | Действие | Результат |
|---|---|---|
| 1 | Загрузка документа из БД по переданным ссылкам | Полный текст и структура документа |
| 2 | Построение чанков и иерархии | Разбиение на семантические фрагменты (не более 512 токенов), построение ltree-иерархии |
| 3 | Вычисление Embeddings | Векторные представления для каждого текстового чанка |
| 4 | Построение векторного индекса | Сохранение чанков, эмбеддингов и индексов в БД |

**Запрос:**
```json
{
  "document_id": "b3a8f1c2-...",
  "registry_doc_id": 42,
  "links": {
    "document": "/api/v1/registry/documents/42",
    "versions": "/api/v1/registry/documents/42/versions"
  },
  "options": {
    "chunk_size": 512,
    "chunk_overlap": 50
  }
}
```

| Поле | Тип | Обязательность | Описание |
|---|---|---|---|
| `document_id` | string | Да | ID документа (UUID) |
| `registry_doc_id` | int | Да | ID документа в реестре |
| `links` | object | Да | Ссылки для загрузки данных из БД |
| `options.chunk_size` | int | Нет | Максимальный размер чанка в токенах (по умолчанию 512) |
| `options.chunk_overlap` | int | Нет | Перекрытие чанков в токенах (по умолчанию 50) |

**Ответ `201`:**
```json
{
  "document_id": "b3a8f1c2-...",
  "status": "completed",
  "indexed_at": "2026-05-15T12:00:18Z",
  "chunks_count": 34,
  "index_stats": {
    "text_chunks": 28,
    "table_chunks": 3,
    "image_chunks": 3,
    "total_embeddings": 31
  }
}
```

| Поле | Тип | Описание |
|---|---|---|
| `document_id` | string | ID документа |
| `status` | string | Статус: `completed`, `failed` |
| `indexed_at` | string | Время завершения индексации |
| `chunks_count` | int | Общее количество созданных чанков |
| `index_stats` | object | Статистика индексации |

### DELETE /rag/index/{document_id}

Удаление всех чанков документа из векторного индекса.

**Ответ `200`:**
```json
{
  "document_id": "b3a8f1c2-...",
  "deleted_count": 128,
  "status": "completed"
}
```

---

### GET /rag/index/{document_id}/status

Статус индексации документа.

**Ответ `200`:**
```json
{
  "document_id": "b3a8f1c2-...",
  "status": "indexed",
  "chunks_count": 34,
  "has_embeddings": true,
  "indexed_at": "2026-05-15T12:00:18Z"
}
```

## Режим B: RAG поиск (читает БД)

Этот режим используется для поиска по проиндексированным документам и генерации ответов.

### POST /rag/search

Гибридный поиск (dense + sparse + pg_trgm) с реранжированием.

**Запрос:**
```json
{
  "query": "ледовый класс Arc4",
  "top_k": 10,
  "filters": {
    "document_type": ["normative"],
    "date_from": "2000-01-01",
    "date_to": "2026-12-31"
  },
  "search_type": "hybrid",
  "rerank": true
}
```

| Поле | Тип | Обязательность | Описание |
|---|---|---|---|
| `query` | string | Да | Поисковый запрос |
| `top_k` | int | Нет | Число результатов (по умолчанию 10) |
| `filters` | object | Нет | Фильтры: `document_type`, `date_from`, `date_to` |
| `search_type` | string | Нет | `hybrid`, `sparse`, `dense` (по умолчанию `hybrid`) |
| `rerank` | bool | Нет | Применять реранжирование (по умолчанию true) |

**Ответ `200`:**
```json
{
  "query": "ледовый класс Arc4",
  "results": [
    {
      "chunk_id": "chk-001",
      "document_id": "doc-norm-001",
      "document_title": "Правила РС",
      "page": 42,
      "text": "Для ледового класса Arc4 толщина обшивки должна быть не менее 12 мм.",
      "score": 0.92,
      "metadata": {
        "heading": "4.2 Требования к обшивке",
        "section": "Ледовые усиления"
      }
    }
  ],
  "search_type_used": "hybrid",
  "processing_time_ms": 120,
  "total_found": 15
}
```

### POST /rag/generate

Генерация ответа LLM с опорой на контекстные чанки.

**Запрос:**
```json
{
  "messages": [
    { "role": "system", "content": "" },
    { "role": "user", "content": "Какая толщина обшивки для Arc4?" }
  ],
  "context_chunks": [
    {
      "chunk_id": "chk-001",
      "text": "Для ледового класса Arc4 толщина обшивки должна быть не менее 12 мм.",
      "document_id": "doc-norm-001",
      "document_title": "Правила РС",
      "page": 42,
      "heading": "4.2 Требования к обшивке"
    }
  ]
}
```

| Поле | Тип | Обязательность | Описание |
|---|---|---|---|
| `messages` | array | Да | Сообщения диалога (system + user) |
| `context_chunks` | array | Да | Контекстные чанки для опоры генеративного движка |

**Ответ `200`:**
```json
{
  "content": "Согласно Правилам классификационного общества, для ледового класса Arc4 толщина обшивки должна быть не менее **12 мм** (источник: «Правила РС», раздел 4.2, стр. 42).",
  "sources": [
    {
      "document_id": "doc-norm-001",
      "document_title": "Правила РС",
      "page": 42,
      "fragment_id": "chk-001",
      "heading": "4.2 Требования к обшивке"
    }
  ]
}
```
