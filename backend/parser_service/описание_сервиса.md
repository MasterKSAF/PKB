### **3\. Логика работы файлов и взаимодействия**

* Эндпоинт `/process`: создаёт `TaskInfo`, добавляет в `TaskStore`. Фоновая задача запускает `Pipeline` с шагами:  
  * `DownloadStep` – скачивает файл из MinIO, сохраняет `original_file_name`.  
  * `ValidateStep` – проверка размера, MIME, безопасности.  
  * `ParseStep` – получает парсер через `ParserFactory`, вызывает `parse()`, возвращает `ParseResult`.  
  * `NormalizeStep` – загружает изображения в MinIO, заменяет `data` на `image_key`, строит JSON-контейнер.  
  * `StandardizeStep` – преобразует контейнер в целевой формат (`document.source`, `document.block`, `quality`).  
  * `StoreResultStep` – сохраняет финальный JSON и HTML в `TaskStore`, переводит статус в `COMPLETED`.  
* `/preview`: аналогичный пайплайн, но перед `ParseStep` вставляется `TruncatePdfStep` (обрезание до `max_pages` страниц). Ответ содержит только `document.source` (общее число страниц из оригинального PDF) и метаданные.  
* `/status`: использует `TaskStore.wait_for_change` (longpoll через `asyncio.Condition`) и возвращает `StatusResponse`.  
* `/result`: возвращает `task_info.result` после завершения.  
* `/processes`: возвращает список активных задач из `TaskStore`.


### **5\. Подробная документация сервиса**

Parser Service – внутренний микросервис для извлечения текста, таблиц и изображений из PDF (с поддержкой DOC/DOCX в виде заглушек).  
Вход: ссылка на файл в MinIO (`file_key`).  
Выход: JSON‑контейнер со структурированным содержимым.  
Особенности: не имеет доступа к БД, состояние задач хранится в памяти (TTL 7 дней), longpoll на 15 секунд.

#### Эндпоинты

| Метод | URL | Описание |
| :---- | :---- | :---- |
| POST | `http://127.0.0.1:8087/api/v1/parser/process` | Асинхронный запуск обработки. Возвращает `task_id` и статус `accepted`. |
| POST | `http://127.0.0.1:8087/api/v1/parser/preview` | Синхронный предпросмотр первых `max_pages` страниц. Не сохраняет изображения. |
| GET | `http://127.0.0.1:8087/api/v1/parser/process/{task_id}/status` | Longpoll (до 15 сек) – возвращает текущий статус, прогресс, кол‑во страниц. |
| GET | `http://127.0.0.1:8087/api/v1/parser/process/{task_id}/result` | Получение итогового JSON (после завершения). |
| GET | `http://127.0.0.1:8087/api/v1/parser/processes` | Список активных задач (статусы `accepted`/`processing`). |
|  | `http://127.0.0.1:8087/docs#/` | документация сервиса |

#### Форматы запросов/ответов.

#### Коды ошибок (в поле `error.code`)

`FILE_NOT_FOUND` (404) – файл отсутствует в MinIO.  
`FILE_TOO_LARGE` (413) – превышен размер (500 MB) или число страниц (\>2000).  
`UNSUPPORTED_FORMAT` (415) – MIME не PDF/DOC/DOCX.  
`PARSER_FAILED` (500) – критическая ошибка парсинга.  
`STORAGE_ERROR` (502) – ошибка доступа к MinIO.  
`TASK_NOT_FOUND` (404) – задача не найдена или удалена.  
`TASK_EXPIRED` (410) – результат удалён по TTL.  
`VALIDATION_ERROR` (422) – ошибка валидации входных данных.  
`INTERNAL_SERVER_ERROR` (500) – необработанное исключение.
